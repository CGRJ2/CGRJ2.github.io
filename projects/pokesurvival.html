네트워크 시스템

Photon AppId 분리 + Firebase 연동으로 io 게임의 즉시 접속성과 높은 플레이 밀도를 확보한 멀티서버 구조

설계 배경 — io 게임의 핵심 가치: 즉시 접속성

기존 방식의 한계
  • 단일 AppId에서 Room 단위로 CCU 분할 사용
  • 로비 대기 유저도 CCU를 점유 → 실제 플레이 인원 제한
  • 서버 가득 참 → "잠시 후 재시도" UX → io 게임 특성과 맞지 않음
해결 전략
  • Photon AppId 자체를 분리해 CCU를 물리적으로 분산
  • Lobby Server / InGame Server CCU가 서로 독립 운영
  • 접속 시 인원이 가장 많은 서버 자동 선택 (플레이 밀도 압축)

서버 구조

Photon (실시간 게임 담당) ├─ Lobby Server (로그인, 파티 대기 공간) │ └─ CCU 독립 운영 — 로비 인원이 인게임에 영향 없음 └─ InGame Server (실제 게임 진행) └─ JoinOrCreateRoom("UniversalRoom") — 단일 룸으로 즉시 합류 Firebase RTDB (서버 메타 / 조정 계층) └─ ServerData { id, type, maxPlayerCount, curPlayerCount, reservedPlayerCount } └─ 클라이언트가 Firebase 조회 → 최적 서버 결정 → Photon AppId 교체 → 연결

서버 전환 메커니즘

기본 전환 순서
Disconnect → AppId 갱신 → ConnectUsingSettings
서버 이동 전 커스텀 프로퍼티 백업 후 재연결 시 복원
예약 기반 전환
파티 동시 이동 보장
Firebase RTDB의 reservedPlayerCount를 트랜잭션으로 증가시켜 자리 선점 후 이동
최적 서버 선택
"접속 가능한 서버 중 현재 인원 최다"
플레이 밀도를 유지하는 io 게임 특성에 최적화

유저 데이터 관리

게스트/로그인 유저를 단일 플레이 흐름으로 처리하는 Photon + Firebase 이중 레이어 구조

데이터 책임 분리

Photon (실시간 데이터)
  • 인게임 플레이 데이터 (레벨, 재화, 킬수, 아이템)
  • 실시간 네트워크 동기화 대상
  • 서버 이동 시에도 CustomProperties로 즉시 복원
  • 게스트 유저는 앱 종료 시 휘발
Firebase (영속 데이터)
  • 계정 기반 영속 데이터 (닉네임, 누적 통계, 랭킹)
  • 로그인 유저만 저장 — 로그인 시 CustomProperties와 동기화
  • 종료/갱신 시점에 주요 데이터 저장

유저 유형별 흐름

게스트 유저
로그인 없이 즉시 플레이. Photon CustomProperties만 사용. DB 부하 없음. 앱 종료 시 데이터 휘발
로그인 유저
Firebase Auth 로그인 → DB에서 UserData 로드 → CustomProperties 동기화. 인게임 로직은 게스트와 동일 흐름

인게임 로직을 CustomProperties 중심으로 통합하여, 게스트/로그인 유저 분기 로직을 최소화하고 단일 흐름으로 처리.

랭킹 시스템

현재 구조
게임 오버 시 전체 랭킹 조회 → 클라이언트 정렬 → Top 10 추출. 유저 수 증가 시 트래픽/정렬 비용 증가
개선 방향 (설계)
DB 쿼리 레벨에서 orderByChild + limitToLast(10) 정렬·제한 조회. 점수 분포 기반 컷라인으로 상위 퍼센트 산출

ScriptableObject 기반 데이터 UI 갱신 구조

포켓몬 데이터가 추가·수정될 때 코드 수정 없이 UI에 자동 반영되는 데이터 중심 UI 설계

구성 흐름

PokemonData (ScriptableObject) ← 데이터 단일 소스 ↓ Panel_PokemonList ← SO 전체 로드, 슬롯 수 자동 관리 (부족 시 생성 / 초과 시 비활성) ↓ Slot_StartingPokemon ← 단일 포켓몬 표현, 클릭 이벤트 전달만 담당 ↓ Panel_PokemonInfo ← 선택된 SO 기반으로 상세 정보 갱신 (도감 번호, 스탯, 타입, 이미지)

주요 특징

🔌

코드 수정 없는 콘텐츠 확장Resources/StartingPokemons 경로에 SO 추가만 하면 UI 자동 반영

→

단방향 데이터 흐름리스트 → 슬롯 → 상세 패널로 데이터가 단방향으로 전달되어 의존성 최소화

🗂️

확장성도감·아이템 리스트·몬스터 리스트 UI에 동일 패턴 적용 가능. Addressables 구조로 쉽게 이전 가능
